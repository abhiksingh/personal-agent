# Security Policy

## Supported Versions

PersonalAgent is currently pre-release. Security fixes are handled on the latest state of `main`; older commits, branches, and forks should not be assumed to receive security patches.

## Reporting a Vulnerability

Please report vulnerabilities privately to `github@abhisheksingh.com`.

When possible, include:

- a concise description of the issue
- affected components or paths
- reproduction steps or a proof of concept
- impact assessment
- any suggested mitigation

Please do not open a public GitHub Issue for undisclosed vulnerabilities, suspected secret exposure, or exploit details.

## What to Expect

- We will aim to acknowledge receipt within 3 business days.
- We will investigate and determine severity and remediation scope as quickly as possible.
- If the report is valid, we will coordinate disclosure timing with the reporter once a fix or mitigation path exists.

## Project Context

This file is the contributor-facing reporting policy. The deeper internal security baseline and guardrails live in [`docs/harness/SECURITY.md`](./docs/harness/SECURITY.md).
